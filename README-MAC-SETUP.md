# Запуск на MacBook (для демонстрации)

Ничего устанавливать не нужно — Node.js уже "зашит" внутрь файла запуска.

## Первый запуск (один раз)

1. Перенеси и распакуй архив на Mac (AirDrop / флешка / облако).
2. Открой **Терминал** (Launchpad → Terminal, либо через Spotlight: ⌘+Пробел → "Terminal").
3. Перетащи распакованную папку `corporate-wheel-mac-demo` прямо в окно Терминала — путь подставится сам. Допиши перед ним `cd ` и нажми Enter:

   ```
   cd [сюда перетянутая папка]
   ```

4. Запусти:

   ```
   bash start-mac.command
   ```

5. Подожди пару секунд — браузер откроется сам на `http://localhost:3100`.

Дальше можно просто **дважды кликнуть на `start-mac.command`** в Finder — после первого запуска через Терминал это уже будет работать как обычный ярлык.

## Перед показом руководству — поправь `.env`

Открой файл `.env` (обычный текстовый файл, можно в TextEdit) и подставь реальные данные:

```
ALLOWED_EMAIL_DOMAINS=company.com      # реальный домен почты компании
COMPANY_NAME=Company                   # реальное название компании
ANNIVERSARY_YEARS=10                   # реальное число лет
SUPER_PRIZE_DESCRIPTION=...            # текст сертификата/приза
FUN_FACTS=факт1|факт2|факт3            # факты о компании через "|"
```

Сейчас там тестовые значения — если не поменять, руководство увидит "Company" и "@company.com". После правки файла просто перезапусти `start-mac.command` (закрой окно Терминала и открой заново).

## Демонстрация и повторные прогоны

Внизу справа на сайте есть кнопка **"🔄 Сбросить тест"** — снимает cookie и очищает записи (email/IP), можно проходить сценарий заново сколько угодно раз во время показа.

## Важно перед показом руководству

- **IP-блокировка**: если показываешь с нескольких устройств в одной Wi-Fi сети, второе устройство может получить "Упс, уже участвовали" из-за общего IP. На время демо можно поставить `ENABLE_IP_LOCK=false` в `.env`.
- Звук требует клика (так устроены браузеры) — появится сразу после нажатия "Крутить колесо", это нормально.
- Чтобы остановить сервер — закрой окно Терминала или нажми `Ctrl+C`.
- Если macOS всё же покажет предупреждение о неизвестном разработчике — выбери "Открыть все равно" (System Settings → Privacy & Security → внизу будет кнопка "Open Anyway").

## Если что-то пошло не так

Если `bash start-mac.command` выдаёт ошибку про "Permission denied" или архитектуру — на Mac есть два бинарника:
- `corporate-wheel-arm64` — для Mac с чипом Apple (M1/M2/M3/M4)
- `corporate-wheel-x64` — для Mac с процессором Intel

Скрипт выбирает нужный автоматически, но если нужно запустить руками:
```
chmod +x corporate-wheel-arm64
./corporate-wheel-arm64
```
(или `corporate-wheel-x64`, если Intel Mac)

## Структура проекта

```
corporate-wheel-arm64, corporate-wheel-x64  — готовые запускаемые файлы (со встроенным Node.js)
start-mac.command                            — скрипт запуска для двойного клика
server/, package.json                        — исходный код (для доработок и пересборки)
public/                                      — фронтенд (HTML/CSS/JS)
data/                                        — entries.json с записями участников
.env                                         — вся настраиваемая часть: домен, тексты, призы, факты
```

## Если нужно внести изменения и пересобрать (для разработчика)

```
npm install
# правишь код в server/ или public/
npm run build:mac
```
Соберёт новые `corporate-wheel-arm64` / `corporate-wheel-x64` в папку `dist/` — их нужно будет положить рядом с `public/`, `.env` и `data/` (как сейчас).
