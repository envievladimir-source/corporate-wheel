# Запуск на MacBook (для демонстрации)

Ничего устанавливать не нужно — Node.js уже "зашит" внутрь файла запуска.

## Первый запуск (один раз)

1. Перенеси и распакуй архив на Mac (AirDrop / флешка / облако).
2. Открой **Терминал** (Launchpad → Terminal, либо через Spotlight: ⌘+Пробел → "Terminal").
3. Перетащи распакованную папку `corporate-wheel-mac-demo` прямо в окно Терминала — путь подставится сам. Допиши перед ним `cd ` и нажми Enter:

   ```
   cd [сюда перетянутая папка]
   ```

4. Запусти:

   ```
   bash start-mac.command
   ```

5. Подожди пару секунд — браузер откроется сам на `http://localhost:3100`.

Дальше можно просто **дважды кликнуть на `start-mac.command`** в Finder — после первого запуска через Терминал это уже будет работать как обычный ярлык.

## Перед показом руководству — перевір `.env`

Дизайн зараз зроблений спеціально під Evoplay (10 років, ДЖЕКПОТ, українська мова). Відкрий файл `.env` (звичайний текстовий файл, можна в TextEdit) і за потреби підстав актуальні дані:

```
ALLOWED_EMAIL_DOMAINS=evoplay.com      # реальний домен пошти компанії
COMPANY_NAME=Evoplay                   # назва компанії (для хвилі в заголовку та плашки "Нам X років")
ANNIVERSARY_YEARS=10                   # реальна кількість років
SUPER_PRIZE_DESCRIPTION=...            # текст під назвою призу в картці результату
FUN_FACTS=факт1|факт2|факт3            # факти в секції "Факти про нас" (до 3 карток)
```

Після правки файлу просто перезапусти `start-mac.command` (закрий вікно Терміналу і відкрий заново).

## Google Sheets: автоматическая запись email + страны (необязательно)

Если хочешь, чтобы каждая регистрация (email и страна, включая вручную вписанную при выборе "Інша") автоматически добавлялась строкой в Google Таблицу — настраивается один раз через сервис-аккаунт Google. Это делается вручную, через твой Google-аккаунт, я не могу сделать это за тебя.

1. Зайди на [console.cloud.google.com](https://console.cloud.google.com), создай (или выбери) проект.
2. В поиске найди **"Google Sheets API"** → нажми **Enable**.
3. Слева: **IAM & Admin → Service Accounts → Create Service Account**. Имя любое, роль можно не назначать — жми Done.
4. Открой созданный service account → вкладка **Keys → Add Key → Create new key → JSON**. Скачается `.json`-файл — он больше никогда не появится повторно, сохрани его.
5. Открой этот `.json`-файл текстовым редактором, найди там два поля: `client_email` и `private_key`.
6. Создай новую Google Таблицу (или используй существующую) — можно сразу добавить первой строкой заголовки: `Час | Email | Країна`.
7. Скопируй **ID таблицы** — это длинная строка в адресной строке между `/d/` и `/edit`:
   `https://docs.google.com/spreadsheets/d/ВОТ_ЭТО_ID/edit`
8. В самой таблице нажми **Поделиться (Share)** и добавь туда `client_email` из шага 5 с правом **Редактор (Editor)**. Без этого шага запись работать не будет.
9. Открой `.env` и заполни:

```
GOOGLE_SHEETS_ID=id_из_шага_7
GOOGLE_SHEETS_RANGE=Sheet1!A:C
GOOGLE_SERVICE_ACCOUNT_EMAIL=client_email_из_json
GOOGLE_PRIVATE_KEY=private_key_из_json
```

`private_key` из JSON-файла выглядит как `"-----BEGIN PRIVATE KEY-----\nMIIEvQ...\n-----END PRIVATE KEY-----\n"` — просто вставь это значение целиком как есть (вместе с `\n`), в кавычках или без, ничего вручную менять не нужно.

Перезапусти `start-mac.command` — с этого момента каждая новая регистрация будет дописываться в таблицу (Время, Email, Страна). Если что-то настроено неверно (ошибка доступа, неверный ID), сайт продолжит работать как обычно — запись в таблицу просто не произойдёт, ошибка попадёт в консоль сервера, а не покажется пользователю.

## Демонстрация и повторные прогоны

Внизу справа на сайте есть кнопка **"🔄 Сбросить тест"** — снимает cookie и очищает записи (email/IP), можно проходить сценарий заново сколько угодно раз во время показа.

## Важно перед показом руководству

- **IP-блокировка**: если показываешь с нескольких устройств в одной Wi-Fi сети, второе устройство может получить "Упс, уже участвовали" из-за общего IP. На время демо можно поставить `ENABLE_IP_LOCK=false` в `.env`.
- Звук требует клика (так устроены браузеры) — появится сразу после нажатия "Крутить колесо", это нормально.
- Чтобы остановить сервер — закрой окно Терминала или нажми `Ctrl+C`.
- Если macOS всё же покажет предупреждение о неизвестном разработчике — выбери "Открыть все равно" (System Settings → Privacy & Security → внизу будет кнопка "Open Anyway").

## Если что-то пошло не так

Если `bash start-mac.command` выдаёт ошибку про "Permission denied" или архитектуру — на Mac есть два бинарника:
- `corporate-wheel-arm64` — для Mac с чипом Apple (M1/M2/M3/M4)
- `corporate-wheel-x64` — для Mac с процессором Intel

Скрипт выбирает нужный автоматически, но если нужно запустить руками:
```
chmod +x corporate-wheel-arm64
./corporate-wheel-arm64
```
(или `corporate-wheel-x64`, если Intel Mac)

## Структура проекта

```
corporate-wheel-arm64, corporate-wheel-x64  — готовые запускаемые файлы (со встроенным Node.js)
start-mac.command                            — скрипт запуска для двойного клика
server/, package.json                        — исходный код (для доработок и пересборки)
public/                                      — фронтенд (HTML/CSS/JS)
data/                                        — entries.json с записями участников
.env                                         — вся настраиваемая часть: домен, тексты, призы, факты
```

## Если нужно внести изменения и пересобрать (для разработчика)

```
npm install
# правишь код в server/ или public/
npm run build:mac
```
Соберёт новые `corporate-wheel-arm64` / `corporate-wheel-x64` в папку `dist/` — их нужно будет положить рядом с `public/`, `.env` и `data/` (как сейчас).
